Below is a handy little Swift Package I built for adding a [SwiftLint](https://github.com/realm/SwiftLint) build phase to Tuist's temporary `Manifests.xcodeproj` project. This allows you to see SwiftLint warnings for your Tuist manifest files while working on them in Xcode!

(This idea could easily be adapted to add any sort of build phase you'd like to said project.)

```swift
import Foundation
import PathKit
import XcodeProj

// This is a simple script that adds a SwiftLint build phase to the `Manifests.xcodeproj` project
// that is generated by Tuist when running `tuist edit --permanent`.

// Get reference to Manifests.xcodeproj
let projectRoot = #fileName.replacingOccurrences(of: "/path/to/TuistManifestProjectSwiftLint/Sources/TuistManifestProjectSwiftLint/main.swift", with: "")
let manifestsProjectPath = Path("\(projectRoot)/Manifests.xcodeproj")
var manifestsProject = try XcodeProj(path: manifestsProjectPath)

// Create SwiftLint build phase
let swiftLintBuildPhase = PBXShellScriptBuildPhase(
	name: "SwiftLint",
	shellPath: "/bin/sh",
	shellScript: "swiftlint --config Tuist/.swiftlint.yml"
)

// Add object at project level to define build phase settings
manifestsProject.pbxproj.add(object: swiftLintBuildPhase)

// Insert reference to build phase as the first step in the `<rootDirectoryName>Manifests` build phases
let rootDirectoryName = (projectRoot as NSString).lastPathComponent
manifestsProject.pbxproj.nativeTargets.forEach {
	if $0.name == "\(rootDirectoryName)Manifests" {
		$0.buildPhases.insert(swiftLintBuildPhase, at: 0)
	}
}

// Write changes to Manifests.xcodeproj
try manifestsProject.write(path: manifestsProjectPath)
```

We run this from the command line as a part of a [`just`](https://github.com/casey/just) recipe to bootstrap the Manifests Xcode project, which looks like this:

```shell
edit:
	# Generating the Tuist Manifests project
	@tuist edit --permanent --only-current-directory

	# Adding the SwiftLint build phase
	@swift run --package-path Tuist/scripts/TuistManifestProjectSwiftLint

	# Opening the Tuist Manifests project
	@xcrun xed Manifests.xcworkspace
```
